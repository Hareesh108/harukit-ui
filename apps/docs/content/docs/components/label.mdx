## Installation

### npm

```bash
npx harukit@latest add label
```

### pnpm

```bash
pnpm dlx harukit@latest add label
```

### yarn

```bash
yarn harukit@latest add label
```

### bun

```bash
bunx --bun harukit@latest add label
```

## Usage

```tsx
import { Label } from "@/components/ui/label"
import { Input } from "@/components/ui/input"

export default function LabelDemo() {
  return (
    <div className="grid w-full max-w-sm items-center gap-1.5">
      <Label htmlFor="email">Email</Label>
      <Input type="email" id="email" placeholder="Enter your email" />
    </div>
  )
}
```

## API Reference

### Label

The main label component.

```tsx
import { Label } from "@/components/ui/label"
```

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `htmlFor` | `string` | - | The ID of the form control that the label is associated with. |
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child, merging their props and behavior. |

### Label Variants

The Label component supports different styling variants through the `labelVariants` function:

```tsx
import { labelVariants } from "@/components/ui/label"

const labelClasses = labelVariants()
```

## Styling

The Label component uses Tailwind CSS classes for styling:

- **Typography**: `text-sm font-medium leading-none`
- **Color**: Inherits from parent or uses `text-foreground`
- **Disabled**: `peer-disabled:cursor-not-allowed peer-disabled:opacity-70`
- **Spacing**: Consistent with form layout

You can customize the appearance by:

1. **Overriding classes**: Add your own Tailwind classes
2. **CSS variables**: Modify the label color variables
3. **Custom variants**: Create your own label variants

## Accessibility

The Label component includes:

- Proper association with form controls via `htmlFor`
- Screen reader support
- Keyboard navigation support
- Disabled state handling
- Required field indicators

### Best Practices

1. **Always use `htmlFor`**: Associate labels with their form controls
2. **Be descriptive**: Use clear, concise text that describes the input
3. **Include required indicators**: Use asterisks or other visual cues for required fields
4. **Group related labels**: Use fieldset and legend for groups of related inputs
5. **Provide context**: Add descriptions when needed for complex inputs

## Dependencies
